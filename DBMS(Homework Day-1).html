Database Management System Theory

Data consists of raw, unprocessed facts and figures without context. Information is data that has been processed, organized, or structured to provide meaning and context. While data is the raw input, information is the meaningful output derived from analyzing and interpreting that data.

Meta Data:

Meta Data is defined as data about data. It is helpful as it helps to keep record of the large number of data that might be in the database

Data dictionary:

Data dictionary is aso similar to meta data, differ in that it is a centralied repository where meta data is stored.

Table:

Table is a part of DBMS by which we can organize the data. The rows usually represent a unique piece of data whereas the columns define an attritube of that data.

Tuple:

A single entry in a table is referred to as a tuple, one single of the table is a tuple.

Field:

A field is a single column in the row. It defines the table's structure by specifying the type of data that can be stored.

Domain:

The range or specific type of data that can be entered into a field is a domain. For 'Age' the maximum value might be kept to 150, where 0 to 150 would be it's domain.

Constrains:

It is a rule that limits the type of data that is permissible in a field. In the 'Age' field, we can make it so that only numeral data types are allowed.

Key:

They're used to idenfity rows or tuples in a table uniquely.
